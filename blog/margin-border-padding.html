Intro

Margin Border and Padding are three core concepts of what is called the 'Box Model'. This is a must-understand for effective web design and layout when building a website. To improve as a web developer, it crucial to have a strong grasp on these concepts. First, we need to make sure we understand what the box model is.

The Box Model

What is the Box Model? When Individual elements are created on a website with HTML, they are organized and arranged as Rectangular boxes. Without rules added to make these visible, it doesn't neccessarily appear that way, but these boxes wrap around every single element on the page. On top of this, elements can also have other boxes inside of them or next to them.


One way to see this in action is to bring up developer tools (right click + inspect on Google Chrome). From there, in the styles section, you can search for "*", to target all elements. From there, you can add a border property "Border 3px solid red;". This draws an outline around the individual elements and lets you see how they are arranged to make the complete design that you as the end user actually interact with. Pretty Neat!


Opening developer tools in Google Chrome Browser to add border styles.

As you can see here, we can apply this as a rule to websites to help illustrate their web element lay out and design.

Picture: Demonstration of the box model on the W3 Schools web page.

Finally, you can actually see the box model on developer tools. I like to think about it from an outside in point of view, from the outermost to the innermost aspect.

The box model uses Margin, Border & Padding to manipulate how the rectangular box is displayed.

Margin: This is space outside of the element. I think of it as distance from other elements, such as a parent box or container.
Border: As the name suggests, this is a border around the outside edges of the element.
Padding: This is extra space added within the element. Almost like wearing a big bulky jacket adds padding to your (human) body :-).
Content: Finally, we have the actual defined content within.

Margin

Margin Properties create space outside of the element, pushing it away from other content. This can be specified in each direction.
For example, if we wanted to create rules to make space 6 pixels from the top, 4 pixels from the right, 10 pixels from the bottom and 8 pixels from the left, we could do that as:
Margin-top: 6px;
Margin-right: 4px;
Margin-bottom: 10px;
Margin-top: 8px;

Alternately, there is a shorthand where you can declare margin once, and specify the rules from clockwise:
Margin: 6px 4px 10px 8px;
This may help you keep your code cleaner and more concise, while providing flexibility!

Border

One step in, we have border. This provides ways to specify the style, width and color of an elements border. Among Other Things!

We have already seen what borders can be used for when examining a page's layout. Of course, there are many other uses for borders, such as adding emphasis, styling or giving an outline to elements. Quite often, during development, I will add temporary borders when positioning or manipulating elements to make it easier to see how they look.

While shorthand is popular using borders, you can also specify to only add a border to one specified side of the element.
One example could be: Border-bottom: 3px solid black;

Padding

Padding is... adding padding within the element. This can expand an item out to give it more presence and give control to setting the size of how items display on a web page. Web Elements such as buttons will frequently add padding to make the button larger than just the specified text and improve useability and accessibility.

Again, you can use shorthand to give padding in all four directions, or you can specify rules for each specific direction. One thing to consider is that if you have a specified element width, adding padding will extend that element out in both directions. If you have a Box that is 300px and padding that is 25 pixels, the result will be a total of 350 pixels (25px on each side);

Outro

While this was a highly simplified overview of the Box Model & Margin, Border and Padding, the best way to get better with these concepts is practice! If you aren't comfortable with these aspects yet, I recommend visiting these topics on either W3Schools, or the Mozilla Developer Web Docs.

After that, you should be well on your way to having a solid understanding and being comfortable using these tools in your own projects going forward!
